# data/generated

`grounded_raw.json` — grounded-вопросы, извлечённые и выверенные по документам
корпуса (папка «Источники информации (parsed)»). Каждый вопрос опирается на
реальные числовые значения, названия документов-источников и, где есть, номера
патентов. Это вход для [`scripts/build_dataset.py`](../../scripts/build_dataset.py),
который вместе с `data/seed_questions.yaml` собирает `data/questions.yaml`.

Формат — плоский (`must_include_any`, `expected_values`, … на верхнем уровне);
при сборке они поднимаются в `gold`, а категория берётся из префикса `id`.

## Как это сгенерировано

Фан-аут по 13 темам × набор документов на тему; на каждую тему — два прохода:
генерация вопросов с числовыми эталонами, затем независимая верификация каждого
значения/источника/патента повторным чтением тех же документов. Из выхода
берётся массив вопросов.

Регенерация не автоматизирована как часть раннера (это разовый оффлайн-шаг над
корпусом, а не часть прогона бенчмарка). Чтобы обновить набор — перегенерируйте
вопросы над корпусом, положите массив сюда как `grounded_raw.json` и пересоберите:

```bash
python -m benchmark.scripts.build_dataset
```

Для ручной правки отдельных вопросов проще редактировать итоговый
`data/questions.yaml` напрямую — формат тот же.
